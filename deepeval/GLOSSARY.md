# DeepEval Glossary

## Core Concepts

**LLM Evaluation** - The systematic process of measuring the quality, accuracy, and performance of Large Language Model outputs, as opposed to intuitive assessment ("does this look good?").

**Test Case** - A single unit of evaluation containing an input, actual output (from your LLM), and optional expected output or retrieval context.

**Metric** - A quantitative measure (scored 0-1) used to evaluate a specific aspect of LLM output quality (e.g., relevance, faithfulness, hallucination).

**Threshold** - The minimum score (0-1) a metric must achieve for a test case to pass.

**Golden Dataset** - A curated collection of test cases with known expected outputs, used as the ground truth for evaluation.

## Evaluation Approaches

**End-to-End Evaluation** - Treating the entire LLM application as a black box, evaluating only the final output.

**Component-Level Evaluation** - Evaluating individual parts of an LLM pipeline separately (e.g., retrieval quality vs generation quality).

**LLM-as-a-Judge** - Using an LLM (like GPT-4) to evaluate another LLM's output based on criteria.

## RAG-Specific Terms

**RAG (Retrieval-Augmented Generation)** - A pattern where an LLM generates answers based on retrieved documents from a knowledge base, rather than relying solely on parametric knowledge.

**Retrieval Context** - The documents or text chunks retrieved from a vector database and provided to the LLM as context for generation.

**Vector Database (VectorDB)** - A database optimized for storing and querying high-dimensional vectors (embeddings), used in RAG systems for semantic search.

**Faithfulness** - A metric measuring whether the LLM's output is factually grounded in the provided retrieval context (i.e., no hallucinations).

**Answer Relevancy** - A metric measuring how relevant the LLM's output is to the user's input question.

**Contextual Recall** - A metric measuring whether the retrieved context contains all information needed to produce the expected answer.

**Contextual Precision** - A metric measuring whether the most relevant retrieved documents are ranked higher than less relevant ones.

**Contextual Relevancy** - A metric measuring the overall relevance of all retrieved documents to the input question.

## Evaluation Metrics Categories

**G-Eval** - A research-backed, general-purpose metric that can evaluate on any custom criteria with human-like accuracy using LLM-as-a-judge.

**Hallucination Detection** - Identifying when an LLM fabricates information not present in the source context.

**Task Completion** - For agent systems, measuring whether the agent successfully accomplished its intended goal.

**Bias Detection** - Identifying gender, racial, or political bias in LLM outputs.

**Toxicity** - Measuring harmful, offensive, or inappropriate content in LLM outputs.

## Platform Terms

**Confident AI** - The commercial platform that integrates with DeepEval for team collaboration, dataset management, and production monitoring.

**Synthetic Dataset** - Test data automatically generated by DeepEval for evaluation purposes.

## Domain-Specific (Container Shipping)

**Operational Data** - Daily metrics and information from ships including routes, schedules, cargo counts, fuel consumption, etc.

**Temporal Query** - Questions involving time references ("yesterday", "last week") which require accurate date handling.

**Numerical Precision** - The requirement that numbers (cargo counts, fuel consumption) must be exact, not approximated.

**Structured Data Extraction** - The process of converting PDF tables and documents into embeddings while preserving accuracy.
